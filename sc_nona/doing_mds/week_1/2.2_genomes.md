# HipSci Donor Genomes for Cuomo et al. (125 iPSC lines) — Access Log

**Goal:** Per-donor, base-pair-resolution personalised reference FASTA for each of the 125 HipSci donors in the Cuomo et al. 2020 endoderm scRNA-seq dataset (PRJEB14362), to enable allele-aware alignment of Smart-seq2 reads.

---

## Summary

**TL;DR — everything you need is open access on the EBI FTP, no application required.**

HipSci has performed 30x **PCR-free WGS on HiSeq X Ten** for the iPSC lines (and matched primary fibroblasts) used in Cuomo et al. The variant calls and raw reads are released on **ENA / `ftp.hipsci.ebi.ac.uk`** under PUBLIC status, against **GRCh37.p13** (specifically the 1000G `hs37d5` reference). The relevant deliverables:

| Asset | Where | Access | Format |
|---|---|---|---|
| Per-donor WGS gVCF (530 samples, ~10 GB each) | ENA `PRJEB15299` / `ERP017015` | OPEN (PUBLIC on ENA) | bgzipped VCF + tabix, GATK HaplotypeCaller against GRCh37 |
| Per-donor WGS CRAMs (raw read alignments) | ENA `PRJEB15299` runs `ERR1858xxx`, `ERR1882xxx` | OPEN | CRAM against `hs37d5` |
| Open-access joint multi-sample BCF, 329 OA iPSC lines, imputed+phased from genotype chip | `ftp.hipsci.ebi.ac.uk/hipsci/ftp/data/vep_openaccess_bcf/chr{1..22,X,Y,MT}.bcf` | OPEN | Per-chromosome bgzf BCF (chr1 ≈ 706 MB, chr22 ≈ 133 MB; ~6 GB total) |
| Genotype chip data (Illumina HumanCoreExome-12) | ENA `PRJEB11750` (Infinium HumanExome BeadChip) + the imputed BCFs above | OPEN | VCF |
| `hs37d5.fa.gz` — exact reference used by HipSci pipelines | `ftp.hipsci.ebi.ac.uk/hipsci/ftp/reference/hs37d5.fa.gz` | OPEN | gzipped FASTA |
| Managed-access cohorts (rare disease lines: Bardet–Biedl, monogenic diabetes, Alport, …) | EGA: `EGAD0000100xxxx` (see archive_datasets/) | Controlled — HipSci DAC application | — |

**Important nuance:** WGS does NOT exist for 100% of the Cuomo donors. The WGS programme covered 446 PCR-free WGS samples from 204 individuals (Sanger/HipSci stats); per-sample gVCFs total 530 (204 fibroblast "pf" + 326 iPSC "i"). For donors lacking WGS, you only have **chip-genotype + 1000G/UK10K imputed VCFs** — covering only ~6–10M common variants vs. ~4–5M heterozygous + many rare variants per WGS. For a DNA-language-model project the WGS lines are strongly preferred; donors covered only by the imputed chip data will systematically be missing rare variants in their personalised reference.

**Reference genome:** GRCh37 (assembly `GCA_000001405.1`, in practice the 1000G `hs37d5` build with decoys + EBV + chrMT rCRS). Not GRCh38. If you want GRCh38, you must liftover or re-call from CRAMs.

**Pipeline for VCF → personalised FASTA:** use `bcftools consensus` for haploid/diploid SNP+indel substitution, or `vcf2diploid` if you want two haplotype FASTAs and a chain file for liftover of annotations. See pipeline section below.

## HipSci genomics data: what exists and where

### 1. Per-sample WGS gVCFs (GATK HaplotypeCaller, GRCh37) — **the right primary source**

- **ENA study:** `PRJEB15299` (secondary: `ERP017015`), title *HipSci___Whole_genome_sequencing___healthy_volunteers*.
- **Coverage:** 530 GATK HC per-sample gVCFs covering 204 primary fibroblast (`pf`) samples (germline) + 326 derived iPSC (`i`) samples.
- **Naming:** `HPSI<MMYY><src>-<name>[_<clone>]` — e.g. `HPSI0814pf-doao` is the donor fibroblast from August 2014, donor "doao". The matching iPSC clone(s) for the same donor share the `doao` stem and have the `i-` prefix (e.g. `HPSI0814i-doao_1`).
- **Recommendation:** Use the **fibroblast (`pf`)** VCF where available — it's the germline genome, free of iPSC reprogramming-induced somatic mutations. Fall back to one iPSC clone (`HPSI....i-NAME_N`) for the ~~25 donors with only iPSC WGS.
- **Caller / reference:** GATK HaplotypeCaller against `GRCh37` (assembly accession `GCA_000001405.1`, reference filename `hs37d5.fa.gz`). gVCF format is per-base genotype likelihoods (large: ~9–10 GB per sample).
- **Public status:** all 530 are `PUBLIC` on ENA. Confirmed with `count?query=...AND status="PUBLIC"`.
- **Download URL pattern:**
  `https://ftp.sra.ebi.ac.uk/vol1/analysis/ERZxxx/ERZxxxxxx/<HPSI-line>.wgs.gatk.haplotype_caller.<date>.genotypes.vcf.gz`
- **Full manifest** (one-shot, mapping line ID ↔ ERZ ↔ BioSample ↔ fibroblast/iPSC):
  ```bash
  curl -s 'https://ftp.hipsci.ebi.ac.uk/hipsci/ftp/archive_datasets/ENA.ERP017015.wgs.normals.analysis_files.tsv' \
       > hipsci_wgs_analysis_manifest.tsv
  # OR via ENA API:
  curl -s 'https://www.ebi.ac.uk/ena/portal/api/filereport?accession=PRJEB15299&result=analysis&fields=analysis_accession,sample_accession,sample_alias,submitted_ftp&format=tsv&limit=600'
  ```

### 2. WGS raw reads (CRAMs against `hs37d5`)

- 3,374 runs across `PRJEB15299` + `PRJEB13436` (`ERP013436` — *HipSci_30x_whole_genome_sequencing_of_iPSC_reference_set*).
- Format: CRAM (with `.crai` index), aligned to `hs37d5`, ~30x coverage, HiSeq X Ten paired-end, **PCR-free** libraries.
- All open access via `https://ftp.sra.ebi.ac.uk/vol1/run/ERR.../`.
- Use if you want to re-call variants (e.g. to lift to GRCh38 yourself, or to harmonise with a joint VCF call using GATK GenotypeGVCFs, or to derive structural variants).

### 3. Open-access joint multi-sample BCF (imputed chip genotypes)

- Path: `https://ftp.hipsci.ebi.ac.uk/hipsci/ftp/data/vep_openaccess_bcf/chr{1..22,X,Y,MT}.bcf` (plus `.csi` indexes).
- 329 unique open-access iPSC lines, **chip-genotyped + imputed against 1000G phase 3 / UK10K, phased with SHAPEIT2 + IMPUTE2**.
- Format note: this is the *imputed genotype-array* product, not the WGS callset. So it has only ~6–10M common variants per sample, missing rare alleles and many small indels that the WGS gVCFs have.
- VEP-annotated (Ensembl 93, GENCODE 19, GRCh37.p13).
- README: `ftp.hipsci.ebi.ac.uk/hipsci/ftp/data/vep_openaccess_bcf/README_vep_openaccess_bcf_20180904.txt`
- Total size ~6 GB across all chromosomes.
- **When to use this instead of the per-sample WGS gVCFs:** if you only need common-variant phasing (e.g. for ASE phasing or for population structure), or to cover donors that lack WGS.

### 4. Reference genome — `hs37d5.fa.gz`

- Canonical HipSci reference: `https://ftp.hipsci.ebi.ac.uk/hipsci/ftp/reference/hs37d5.fa.gz`
- Contents (from README): GRCh37 primary chromosomes + unlocalized/unplaced contigs + rCRS mitochondrion (`NC_012920`) + EBV (HHV-4 type 1, `NC_007605`) + a decoy sequence derived from HuRef, BAC/fosmid clones, and NA12878.
- Use this exact FASTA as your base for `bcftools consensus` — the contig names in the WGS VCFs (`1, 2, ..., 22, X, Y, MT`) match this build, with no `chr` prefix.

### 5. Genotype chip data

- ENA `PRJEB11750` — *HipSci genotyping array data for healthy volunteers* — 401 per-sample chip VCFs (Infinium HumanExome BeadChip). This is essentially the input to the imputation pipeline above.
- Less useful than (1) or (3) for our purposes.

### 6. Managed-access cohorts (rare-disease lines) — **probably irrelevant for Cuomo**

- The Cuomo paper uses **125 healthy donors from the NIHR Cambridge BioResource** (Methods, PMC7010688). These are openly consented; their WGS is in `PRJEB15299` PUBLIC.
- Rare-disease lines (Bardet–Biedl, Alport, Kabuki, monogenic diabetes, etc.) are under EGA managed access (`EGAD00001003514`–`EGAD00001003528` for exome-seq, `EGAD00001003529`–`EGAD00001003543` for RNA-seq). Apply via the HipSci Data Access Committee through EGA if you ever need those.

## Mapping Cuomo donors → HipSci lines

The Cuomo paper (PMC7010688, Supplementary Data 2) lists demographics for the 125 donors / 126 lines. The names follow the standard HipSci convention (`HPSIyyyyi-NAME[_clone]`). You need this mapping to pick the right VCFs.

**How to get the donor list:**

Option A — from the run manifest itself: the ENA `PRJEB14362` Smart-seq2 records have a `sample_alias` / `library_name` field that contains the HipSci line. Pull it via:

```bash
curl -s "https://www.ebi.ac.uk/ena/portal/api/filereport?accession=PRJEB14362&result=read_run&fields=run_accession,sample_accession,sample_alias,library_name,experiment_alias&format=tsv&limit=46500" \
     | awk -F'\t' 'NR>1 {print $4}' | grep -oE 'HPSI[0-9]+i-[a-z]+' | sort -u
```

Option B — Zenodo `cell_metadata_cols.tsv` (already downloaded in your data plan) has a `donor` column.

Then intersect that set with the WGS manifest:

```bash
# Save WGS manifest
curl -s 'https://ftp.hipsci.ebi.ac.uk/hipsci/ftp/archive_datasets/ENA.ERP017015.wgs.normals.analysis_files.tsv' \
     | awk -F'\t' 'NR>5 {print $3"\t"$1}' > hipsci_wgs_lines.tsv  # cell_line<TAB>FTP_URL

# Strip clone numbers if needed, since the Smart-seq2 mostly identifies donors not specific clones
# Then per-donor: prefer the `pf` (fibroblast) VCF; fall back to any `i-NAME_N`
```

The expectation: of the 125 Cuomo donors, **the vast majority will have a `pf` fibroblast VCF in `ERP017015`** (this is the standard HipSci germline product). For the residual donors with no pf VCF, you have several options: pick an `i-` iPSC VCF; or use the imputed BCF (`vep_openaccess_bcf`) which has 329 iPSC lines, mostly overlapping; or call your own germline variants from the WGS CRAMs.

## Pipeline: VCF → personalised FASTA

Cuomo et al. themselves **did NOT build per-donor reference FASTAs.** They aligned all cells to a single GRCh37+ERCC reference with STAR (2-pass), called variants per cell with GATK HC at known sites, and used `cardelino` to deconvolve cell → donor by matching read genotypes to the HipSci chip/imputed VCFs. So the published pipeline doesn't help us directly — we need to build personalised refs ourselves.

### Option A — `bcftools consensus` (simplest, haploid SNP+indel substitution)

```bash
# Inputs
WGS_VCF=HPSI0814pf-doao.wgs.gatk.haplotype_caller.20161201.genotypes.vcf.gz
REF=hs37d5.fa            # bgzipped is fine; need .fai
DONOR=HPSI0814pf-doao
OUT=${DONOR}.personalised.fa

# 1. Filter to high-confidence biallelic SNVs+small indels for this single sample
bcftools view -s ${DONOR} -f PASS -m2 -M2 -e 'GT="./." || GT="0/0"' ${WGS_VCF} -Oz -o ${DONOR}.calls.vcf.gz
bcftools index -t ${DONOR}.calls.vcf.gz

# 2. Apply variants to reference (haploid consensus — picks one allele per heterozygous site)
bcftools consensus -f ${REF} ${DONOR}.calls.vcf.gz > ${OUT}

# Index for STAR
samtools faidx ${OUT}
```

Caveats:
- Output is a single haplotype (haploid). For STAR alignment of bulk-ish or Smart-seq2 reads this is usually fine — heterozygous sites get one allele in the ref and reads with the other allele simply mismatch by 1bp. But this hides true heterozygosity and gives biased ASE.
- If you want to mark heterozygous sites in the FASTA with IUPAC ambiguity codes (so STAR's mismatch tolerance treats both alleles as the ref), add `-I/--iupac-codes`.
- Indels shift coordinates ⇒ the personalised FASTA is *not* coordinate-compatible with GENCODE v19 GTF. You need a chain file or you must liftover the GTF. `bcftools consensus -c CHAIN.txt` writes a chain file for exactly this purpose.

### Option B — `vcf2diploid` (true diploid — two FASTAs per donor)

Builds maternal/paternal FASTAs plus chain files. Requires phased VCF input. The HipSci imputed BCFs are phased (SHAPEIT2); the per-sample WGS gVCFs are not — you'd need to phase them first (e.g. with SHAPEIT5 against a reference panel, or use the impute2-phased haplotypes from the open-access joint BCF for common variants).

```bash
java -jar vcf2diploid.jar -id HPSI0814pf-doao -chr hs37d5.fa -vcf donor.phased.vcf.gz -outDir ./doao/
```

Produces `1_maternal.fa`, `1_paternal.fa`, …, `mat.chain`, `pat.chain`. Then build a *combined* diploid STAR index (concat both, with renamed contigs like `1_mat`, `1_pat`) — this lets reads align competitively to both haplotypes and gives clean allele-specific counts.

### Option C — `g2gtools` (Churchill lab)

The cleanest pipeline for ASE-aware alignment. Builds a diploid genome + transformed GTF + chain file, all in one. Originally for mouse Collaborative Cross / DO panels but works for any phased diploid VCF. Use this if you want allele-resolved Smart-seq2 transcript counts downstream.

### Option D — GATK `FastaAlternateReferenceMaker`

Equivalent to `bcftools consensus` but slower and more memory-heavy. Use only if you're already in a GATK pipeline.

### Option E — SNPsplit / WASP-style (no personalised FASTA)

Don't build a personalised reference at all — align to plain GRCh37, then mask reads at known het sites and re-align (WASP), or sort reads to haplotypes post-alignment with SNPsplit using the donor's phased VCF. Used heavily for allele-specific expression studies on hybrid mice and on HipSci-like cohorts. Lower disk footprint (no 125 × 3 GB FASTAs) but you keep an alignment reference bias.

### Recommended for the DNA-LM project

Given the project wants base-pair-resolution coverage tracks that reflect the donor's actual genome:

1. **Use the WGS per-sample gVCFs** (`ERP017015`) for donors that have them, falling back to imputed joint BCF for the rest.
2. **Pre-phase** all per-sample VCFs (SHAPEIT5 against 1000G phase 3 + UK10K, or use Eagle2; or just pull phased haps from `vep_openaccess_bcf` for the common variants that overlap).
3. **Build diploid references** with `vcf2diploid` or `g2gtools`, producing `mat.fa` + `pat.fa` + chains.
4. **Align Smart-seq2 reads** to each donor's diploid STAR index. This naturally gives you base-pair coverage on the donor's own sequence rather than on GRCh37.
5. **Convert per-donor coverage tracks** back to a common coordinate system if needed for cross-donor MSA-style training (`CrossMap` + the chain files).

Disk budget: 125 donors × ~6 GB (diploid FASTA + STAR index) ≈ **~750 GB** for the indexes alone. Doable but plan storage.

## Recommended next steps

1. **Pull the donor list** from `cell_metadata_cols.tsv` (Zenodo) or from `PRJEB14362` ENA metadata. Make a list of the 125 `HPSI...i-NAME` IDs and their corresponding donor stems.
2. **Pull the WGS analysis manifest** (`ENA.ERP017015.wgs.normals.analysis_files.tsv`) and join on donor stem. For each donor, prefer the `pf` (fibroblast) row; fall back to any `i-` row; fall back to the imputed joint BCF for residual donors.
3. **Download just the chromosomes / regions you need** to start. Each full per-sample gVCF is ~10 GB but most positions are HOM-REF; you can pre-filter with `bcftools view -f PASS -e 'GT="./." || GT="0/0"'` to ~500 MB of actual variant calls.
4. **Install `bcftools`** in the project venv (`uv pip install pysam` gives you the Python API; for the CLI use a conda env or `brew install bcftools` on macOS).
5. **Download the reference once:**
   ```bash
   curl -O https://ftp.hipsci.ebi.ac.uk/hipsci/ftp/reference/hs37d5.fa.gz
   gunzip hs37d5.fa.gz && samtools faidx hs37d5.fa
   ```
6. **Sanity-check with one donor end-to-end** — build the personalised FASTA, STAR-index it, align one Cuomo Smart-seq2 cell, check that het-site reads map cleanly without the GRCh37 reference bias.
