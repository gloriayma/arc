# AlphaFold2: architectural details, with focus on column attention

This document summarizes AlphaFold2's architecture, written for a visualization website in the same style as `5.3_gpn_arch_research.md`. The focal point is **column-wise attention** in the Evoformer — the attention module that, at each MSA column (residue position), attends across the rows (sequences). This is the closest direct analog to "vertical attention across cells at the same genomic bin" in our planned scRNA-conditioned DNA LM (see `planning_mds/7_axial_attention_ntv3_integration.md`, `doing_mds/6.1_variable_depth_strategies.md`).

Citations and uncertainties are flagged at the end of each section.

Sources used:
- Jumper et al. 2021, "Highly accurate protein structure prediction with AlphaFold", *Nature* 596:583–589. doi: 10.1038/s41586-021-03819-2
- AlphaFold2 Supplementary Information (~62 pp.) — algorithm pseudocode numbered in this doc as "Suppl. Alg. N"
- DeepMind AlphaFold repo: https://github.com/google-deepmind/alphafold (`alphafold/model/modules.py`, `alphafold/model/config.py`)
- Rao et al. 2021, "MSA Transformer", ICML 2021 — the direct algorithmic precursor of AF2's row+column attention. arXiv:2102.13350

**Scope note: this document is AlphaFold2 only.** Every algorithm number, channel width, feature schema, and head count below refers to AF2 (Jumper et al. 2021). AlphaFold3 (Abramson et al. 2024, *Nature*) replaced the Evoformer with the Pairformer, shrank the MSA module dramatically, and changed the input feature schemas (target features, MSA encoding, template handling are all different). Where AF3 differs on a point that matters for our scRNA mapping, I flag it with "**[AF3 diverges:]**". Otherwise assume "AF2".

## 1. Bottom line on column attention

1. **Where it lives.** The Evoformer block has two attention modules along the MSA axes: (a) **row-wise** gated self-attention along residues, biased by the pair representation, and (b) **column-wise** gated self-attention along sequences. Column attention is module (b). Each Evoformer block runs them in sequence: row attention → column attention → MSA transition → outer-product mean → triangle updates → pair transition. There are **48 Evoformer blocks** (`evoformer_num_block: 48` in the default config).

2. **What it does mechanically.** For each residue position `i` independently, treat the column of `N_seq` MSA tokens at that position as a small sequence and run an 8-head gated self-attention over them. Sequences in the MSA attend to each other at the same position; the residue axis is *not* mixed inside this module. (The residue axis is mixed in row attention and in the pair representation.)

3. **Two variants are used in AF2.**
   - **Full column attention** (`MSAColumnAttention`, Suppl. Alg. 8) in the main Evoformer, which processes the clustered MSA (≤ 512 sequences).
   - **Global column attention** (`MSAColumnGlobalAttention`, Suppl. Alg. 19) in the Extra MSA Stack, which processes a much larger unclustered MSA (≤ 1024 sequences) and uses a pooled-query trick to cut cost from O(N_seq²) to O(N_seq) per column.

4. **No positional encoding on the sequence axis.** The MSA rows are an unordered set; the only "row 0 is special" treatment is that during clustering, row 0 is the *target* (query) sequence, and the cluster-center MSA carries it as row 0. The attention module itself uses no row-position embedding. This is the same design as MSA Transformer (Rao et al. 2021).

5. **Why this module is here.** Co-variation between residues at different columns is captured by the *pair* representation and triangle updates. But co-variation between *sequences* at the same column — e.g. "sequences that have a Cys at position i tend to also have a Cys at position j" — first needs the model to recognize which sequences are similar at each position. Column attention is the per-column information flow that lets a sequence's representation at position `i` depend on what the *other* MSA rows have at position `i`. Combined with row attention and the outer-product-mean → pair update, this is how MSA co-evolutionary signal is extracted.

[CONFIRMED — paper Methods §1.7 + Suppl. §1.6, repo `modules.py:EvoformerIteration`, repo `config.py`]

## 2. Top-level architecture

```
Input MSA (clustered) + Extra MSA + templates + sequence
        │
        ▼
┌─────────────────────────────────────────────────┐
│  Input Embedder                                 │
│   - target_feat, msa_feat, residue_index, …     │
│   - emits initial MSA repr `m`  (s_main, r, c_m)│
│     with c_m = 256                              │
│   - emits initial pair repr   `z`  (r, r, c_z)  │
│     with c_z = 128                              │
└─────────────────────────────────────────────────┘
        │
        ▼
┌─────────────────────────────────────────────────┐
│  Extra MSA Stack ×4 blocks                      │
│   - operates on extra MSA repr `m_extra`        │
│     shape (s_extra, r, c_e),  c_e = 64          │
│   - uses **global** column attention            │
│   - updates pair repr z (outer-product mean,    │
│     triangle updates, etc.)                     │
└─────────────────────────────────────────────────┘
        │
        ▼  (final z from Extra MSA → main Evoformer)
┌─────────────────────────────────────────────────┐
│  Evoformer ×48 blocks                           │
│   - operates on main MSA repr `m`               │
│     shape (s_main, r, c_m)                      │
│   - uses **full** column attention              │
│   - jointly updates `m` and `z`                 │

        │
        ▼
   Single repr (row 0 of `m`) →  Structure Module → 3D coords
```

## 3. The MSA representation — the tensor that column attention acts on

### 3.1 The two tensors at a glance

The Evoformer maintains two representations:

- **MSA representation** `m`, shape `(N_seq, N_res, c_m)`.
- **Pair representation** `z`, shape `(N_res, N_res, c_z)`.

Inside an Evoformer block, `m` and `z` are updated jointly. The MSA repr's two axes have very different meaning:

```
                                    residue axis (size N_res)
                                    ───────────────────────────►
                                    (row attention runs along here)

   sequence axis (size N_seq)      ┌──────────────────────────────┐
       │                           │                              │
       │                           │   m[s, r, :]  ∈  R^{c_m=256} │
       │ (column attention runs    │                              │
       │  vertically here)         │                              │
       │                           │                              │
       ▼                           └──────────────────────────────┘
                                    Each cell is a c_m=256 vector.
                                    Row 0 (s=0) = target / query sequence.
```

Three things this tensor is NOT:

- Not a sparse representation. The full `(N_seq × N_res × 256)` tensor is materialized.
- Not symmetric in the two MSA axes. Row attention is *biased by the pair representation*; column attention is plain self-attention with no bias.
- Not the only place co-variation lives. The pair repr `z` and the outer-product-mean update are also load-bearing.

[CONFIRMED — Suppl. §1.6.1]

### 3.2 How the raw MSA gets reduced to 512 rows (clustering)

A JackHMMER/HHBlits search against UniRef + MGnify + BFD can return tens of thousands of homologs. AF2 caps the main Evoformer input at `N_seq = 512` rows. The reduction is done by **clustering**, not by truncation:

1. **Target is row 0.** The query sequence is always a cluster center and sits at row 0 of the main MSA. The other `N_seq − 1 = 511` cluster centers fill rows 1..511.
2. **Pick the other 511 centers.** During training: a random sample of the deduplicated MSA, re-drawn each step (a form of data augmentation). At inference: deterministic.
3. **Hamming-nearest-neighbor assignment.** Every non-center sequence is assigned to whichever of the `N_seq` centers is closest in Hamming distance over the aligned residues.
4. **Summarize each cluster.** For each cluster, compute a per-residue AA frequency profile and a per-residue mean deletion count across its assigned members. These summaries are packed into the row of the cluster center (see §3.4).
5. **Leftovers → Extra MSA.** Sequences that were not chosen as centers (and do not need to be summarized into a center's profile) are fed as the Extra MSA Stack input, with row count `N_seq_extra ≤ max_extra_msa = 1024`. This is why the Extra MSA Stack exists at all: it lets the model still look at the rest of the alignment, just with a cheaper architecture (`c_e=64`, only 4 blocks, **global** column attention).

**Consequence for column attention.** The `N_seq = 512` rows attended over by column attention are not 512 independent homologs — they are 512 **cluster-summary tokens** (center + profile + deletion stats). When column attention at residue `i` mixes information across rows, it is mixing across *groups* of homologs (each represented by one center + its profile), not individual homologs.

[CONFIRMED — Suppl. §1.2.7 "MSA block deletion and clustering"]

**[AF3 diverges:]** AF3 keeps a much smaller MSA (paper reports MSA depth of order 1024 *total* across main + extra) and does not run a deep MSA-axis stack. The MSA module in AF3 is shallow (4 blocks) and the heavy compute moves to the Pairformer on the pair representation. So the "cluster MSA into 512 centers + run column attention 48 times" pattern is AF2-specific.

### 3.3 `target_feat` — the query sequence as one-hot

`target_feat` is the simplest input AF2 takes: just the user-supplied amino acid sequence, one-hot encoded.

```
target_feat: shape (N_res, 22)

  position i  →  [21-dim one-hot AA  ‖  1 chain-break flag]
```

The 22 channels break down as:

| Channels | Meaning |
|---|---|
| 21 | One-hot over AA categories: 20 standard AAs + `X` (unknown/ambiguous) |
| 1 | `between_segment_residues` — binary chain-break flag, only meaningful for multimers; zero everywhere for a monomer |

Properties:

- **Depends only on the query sequence.** Exists even with zero homologs (single-sequence AF2 runs).
- **No gaps, no mask token.** The query has no gaps in its own coordinate system, and `target_feat` is never masked.
- **Deterministic across training steps.** Unlike `msa_feat`, it does not get re-sampled.
- **Seeds the pair representation.** The initial pair repr is `z[i, j] = LinearLeft(target_feat[i]) + LinearRight(target_feat[j]) + RelPosEnc(j − i)`. So the query sequence is the only thing that initializes `z`; the MSA enters `z` later, through the outer-product-mean update inside each Evoformer block.

[CONFIRMED — Suppl. §1.2.9, Table 1]

**[AF3 diverges:]** AF3 still has a query/target representation but its input embedding handles atom-level tokens (the model can fold nucleic acids and ligands, not just proteins), so the 22-channel one-hot is replaced by a token type embedding with many more categories.

### 3.4 `msa_feat` — the per-row, per-residue MSA feature stack

`msa_feat` is the much richer per-(row, residue) tensor that feeds the 512 rows of the main MSA.

```
msa_feat: shape (N_seq=512, N_res, 49)
```

(Same `N_seq` symbol as the MSA repr `m` in §3.1 — the rows of `msa_feat` are the rows of `m` after input embedding.)

The 49 channels at each `(s, i)` cell are the concatenation of five feature blocks:

| Block | Channels | What it encodes |
|---|---:|---|
| `cluster_msa` | 23 | One-hot of the **cluster center sequence's AA** at position `i`. 23 categories = 20 standard AAs + `X` (unknown) + `-` (gap) + `[MASK]` (BERT-style mask token, only used during training) |
| `cluster_has_deletion` | 1 | Binary: was a residue deleted between position `i` and `i+1` in the original homolog (before MSA gap-aligning)? |
| `cluster_deletion_value` | 1 | Numeric: how many residues were deleted there, passed through `2/π · arctan(d/3)` to squash into [0, 1] |
| `cluster_deletion_mean` | 1 | Numeric: mean of `cluster_deletion_value` **averaged across all sequences assigned to this cluster** (per-cluster scalar) |
| `cluster_profile` | 23 | Per-position **AA frequency distribution** across all sequences assigned to this cluster. Sums to 1 over the 23 categories. This is the "compressed homolog statistics" that makes a cluster row carry more information than just one homolog. |

Total: 23 + 1 + 1 + 1 + 23 = 49. The input embedder projects this 49-dim vector to `c_m = 256` via a single Linear layer: `m[s, i] ← Linear(msa_feat[s, i])`.

**Concrete differences from `target_feat`:**

| | `target_feat` | `msa_feat` |
|---|---|---|
| Shape | `(N_res, 22)` | `(N_seq=512, N_res, 49)` |
| Depends on MSA? | No — pure function of the query | Yes — defined by clustering + each cluster's members |
| Per-row variability | None (one tensor) | 512 different rows |
| Encodes gaps? | No | Yes (`-` is one of the 23 categories) |
| Encodes mask tokens? | No | Yes (`[MASK]` is one of the 23 categories) |
| Encodes deletions? | No | Yes (3 of the 49 channels) |
| Encodes population statistics? | No | Yes (the 23-channel `cluster_profile` is an AA frequency distribution, not a one-hot) |
| Stochastic across training steps? | No | Yes — clustering is re-sampled each step |

[CONFIRMED — Suppl. §1.2.9, Table 1; `alphafold/model/features.py`]

**[AF3 diverges:]** AF3's MSA features are a much smaller schema (the paper shows the MSA module taking a low-dimensional MSA embedding plus a "has deletion" feature, no large cluster profile, no BERT mask). The 49-channel `msa_feat` is AF2-specific.

### 3.5 What privileges row 0

The column-attention module itself is symmetric in the row axis — no positional encoding on `s`, no row-id embedding. The asymmetry between row 0 and rows 1..511 is a **boundary condition** at the encoder's inputs and outputs:

**1. `target_feat` is injected only into row 0 by the input embedder (Suppl. Alg. 3).**

```
m[0, i] ← Linear(msa_feat[0, i]) + Linear(target_feat[i])
m[s, i] ← Linear(msa_feat[s, i])            for s ≥ 1
```

`target_feat` is never broadcast to rows 1..511. Before any attention runs, row 0 has a strictly richer input embedding than the other rows. Column attention preserves and amplifies that distinction — each row has its own Q/K/V projections of its own input vector, so an input difference becomes an output difference.

Note the deliberate redundancy: `msa_feat[0, i]`'s `cluster_msa` block already encodes the query's AA at position `i`, and `target_feat[i]` encodes it again. The two projection matrices are learned independently, so the model can route the "this is the query" signal through whichever it finds more useful. The sum just gives row 0 strictly more input degrees of freedom than rows 1..511.

**2. The structure module reads only row 0.**

After 48 Evoformer blocks, the "single representation" handed to the Structure Module is `m[0, :, :]` projected to `c_s = 384`. The other 511 rows never see the structure loss directly. So during training, the gradient pulling the model toward correct 3D coordinates flows back *through row 0 specifically*. The model learns to route useful information into row 0 because that is where the structure-prediction objective grades it.

Rows 1..511 do see the BERT-style MLM auxiliary loss (`masked_msa_head` predicts the original AA at masked MSA positions across all rows), so they are not gradient-free — but the dominant structural signal is row-0-only.

**3. Recycling reinforces this.**

Between recycling iterations, only the previous iteration's row 0 of `m` (and the full `z`) is added back into the current iteration's input; the rest of the cluster MSA gets re-sampled stochastically. So information that survives across recycling has to either live in row 0 of `m` or in `z`.

**Putting it together for column attention specifically.** The module itself is symmetric in `s`. The asymmetry is enforced at the boundary: row 0 starts different (target_feat injection) and ends different (structure module reads it). Column attention's job is to let the other 511 rows pour information *into* row 0 (and into each other), but the "row 0 is the answer" framing is enforced at the encoder's inputs and outputs, not in the middle.

[CONFIRMED — Suppl. Alg. 3 (InputEmbedder), Suppl. §1.10 (Structure Module input), Suppl. §1.11 (Recycling)]

**[AF3 diverges:]** AF3 doesn't single out "row 0 of the MSA" in the same way because its main computational trunk is the Pairformer over the pair representation, not the Evoformer over the MSA. The query sequence enters AF3 through a "single representation" that bypasses the MSA pipeline; the MSA module is auxiliary. So "row 0 carries the query" is an AF2 framing.

## 4. The Evoformer block (Suppl. Alg. 6)

Each of the 48 Evoformer blocks does the following, in order:

```
                                  ┌─────────────────────┐
   m (N_seq, N_res, c_m)  ───────►│ Row attention       │──┐
                                  │ (pair-biased)       │  │
   z (N_res, N_res, c_z)  ──bias─►│  Suppl. Alg. 7      │  │
                                  └─────────────────────┘  │
                                                            ▼
                                  ┌─────────────────────┐  ⊕ residual
   m  ──────────────────────────► │ Column attention    │──┐
                                  │ Suppl. Alg. 8       │  │
                                  │  (THIS DOCUMENT)    │  │
                                  └─────────────────────┘  │
                                                            ▼
                                  ┌─────────────────────┐  ⊕ residual
   m  ──────────────────────────► │ MSA transition      │──┐
                                  │ (2-layer MLP per    │  │
                                  │  (s, r) token)      │  │
                                  └─────────────────────┘  │
                                                            ▼
                                  ┌─────────────────────┐  ⊕ residual
   m  ──────────────────────────► │ Outer product mean  │──► Δz
                                  │ Suppl. Alg. 10      │
                                  └─────────────────────┘
                                                            ▼
                                                          z update
   z  ──► triangle multiplicative (outgoing) ──► z
       ──► triangle multiplicative (incoming) ──► z
       ──► triangle attention (starting node)  ──► z
       ──► triangle attention (ending node)    ──► z
       ──► pair transition (2-layer MLP)       ──► z
```

(48 such blocks stacked.) The Extra MSA Stack uses the same block but with two differences: `c_m → c_e = 64`, and the column attention module is replaced with **global** column attention.

## 5. Column attention — full deep dive

### 5.1 Full column attention (Suppl. Alg. 8) — main Evoformer

This is the focus of the doc. Its semantics: for each residue position `i = 1..N_res`, attend over the `N_seq` MSA rows at that position with 8-head gated self-attention.

**Pseudocode (paraphrased from Suppl. Alg. 8):**

```
Input  : m  ∈ R^{N_seq × N_res × c_m},  c_m = 256
Output : Δm ∈ R^{N_seq × N_res × c_m}  (additive residual to m)

m ← LayerNorm(m)                                       # per (s,i) vector

# Project to multi-head Q, K, V, and gate.
# H = num_head = 8.  d = c_m / H = 32 (key/value dim per head).
q_{shic} = Linear(m_{si})           shape (N_seq, N_res, H, d)
k_{shic} = Linear(m_{si})           shape (N_seq, N_res, H, d)
v_{shic} = Linear(m_{si})           shape (N_seq, N_res, H, d)
g_{shic} = sigmoid(Linear(m_{si}))  shape (N_seq, N_res, H, d)

# Attention runs along the SEQUENCE axis (s, t), per residue i and head h.
a_{histi} = softmax_t( (1/√d) · Σ_c q_{shic} · k_{thic} )
o_{shic}  = g_{shic} ⊙ Σ_t a_{histi} · v_{thic}

# Concat heads and project back to c_m.
Δm_{si} = Linear_out( concat_h(o_{shic}) )
return Δm
```

**Key properties.**

1. **Axis swap implementation.** In the AF2 codebase, the implementation literally swaps axes so that the sequence dimension becomes the "query length" dimension of a standard attention module, then swaps back:
   ```python
   # alphafold/model/modules.py, MSAColumnAttention.__call__
   msa_act = jnp.swapaxes(msa_act, -2, -3)   # (s, r, c) -> (r, s, c)
   ... LayerNorm + Attention over the "s" axis ...
   msa_act = jnp.swapaxes(msa_act, -2, -3)   # back to (s, r, c)
   ```
   So column attention is "the same module as row attention, but the axes are pre-swapped." The N_res axis becomes the batch axis of the attention call; the N_seq axis becomes its query/key axis.
2. **No pair bias.** Unlike row attention (Suppl. Alg. 7), column attention has *no* bias term from the pair representation. It is plain dot-product attention plus the gating term. The pair repr only injects into the row-attention axis.
3. **Gated.** The output is multiplied element-wise by a sigmoid-gated projection of the input. This is the "gated multi-head attention" used everywhere in AF2 (row, column, triangle attention) and is the same construction as Vaswani 2017 + sigmoid gate.
4. **No positional encoding on the sequence axis.** The 8 heads are not RoPE'd, not sinusoidally encoded — sequences are an unordered set. Row 0 (the target) is distinguished only through learned embeddings inserted during input featurization, not via position-id.
5. **Heads = 8, head dim d = 32.** From `config.py`: `num_head: 8`, and the per-head dim is c_m / H = 256 / 8 = 32. The gate also has shape (H, d).
6. **Mask handling.** The MSA mask `(N_seq, N_res)` is reshaped/broadcast to mask out padded sequences. Padded rows do not contribute keys/values.

**Cost.** For one block: `O(N_seq² · N_res · c_m)` for the attention scores + `O(N_seq · N_res · c_m²)` for the projections. With `N_seq = 512, N_res = 256, c_m = 256` and 48 blocks, this is the dominant cost of the Evoformer alongside triangle attention.

**Information flow.** After this module, the representation at `(s, i)` has been updated by a content-weighted mixture of the other rows' representations at the *same* column `i`. The model can use this to e.g. detect that "rows 5, 12, 18 look like distant homologs of the target at residue i" and adjust accordingly.

[CONFIRMED — Suppl. Alg. 8, repo `modules.py:MSAColumnAttention`, `modules.py:Attention`]

### 5.2 Why column attention is the *interesting* axis for our scRNA work

The 5.3 doc explained that GPN-MSA does **not** have anything like column attention — its 89 non-focal species at each position are flattened to a 445-dim concatenated one-hot and the model is a 1D transformer on the residue axis only. GPN-Star *partially* recovers a column-like mechanism via clade-pooled cross-attention but does not literally do column self-attention.

AF2 (and its parent MSA Transformer, Rao et al. 2021) is the canonical example where:

- The MSA is kept as a true 2-D tensor `(N_seq, N_res, c)` throughout the encoder.
- Mixing along each axis is done by a *separate* attention module.
- The two axes are *not* tied (different Q/K/V matrices, different head counts conceptually possible).

This is the design we're proposing for cells × bp in our scRNA stack. The mapping is:

| AlphaFold2 (MSA) | Our scRNA stack | Notes |
|---|---|---|
| `N_seq` MSA rows | `D` cells (up to 100) | Cells are the rows. |
| `N_res` residue columns | `T` 128-bp bins along the genome window | Bins are the columns. |
| `c_m = 256` MSA channels | `d_cell = 256` per-cell features | Match by coincidence + convention. |
| Row attention (pair-biased) along `N_res` | Row attention along `T` (with RoPE) | Genome-axis attention per cell. |
| Column attention along `N_seq` | Column attention along `D` (no PE) | Cell-axis attention per bin. |
| 48 Evoformer blocks | `N1 = 4` cell-mixing blocks | Many fewer because our cell-mixing module *feeds* a heavy genome trunk (NTv3), it does not *replace* it. |
| Pair representation `z` over residues | (no analog in v0) | Could be added later as an explicit bin × bin tensor for promoter-enhancer relationships. |
| Outer product mean | (no analog in v0) | This is what would build a bin × bin pair tensor *from* the cells × bins MSA. Skipped for v0. |

The structural difference between full column attention (AF2 main Evoformer) and global column attention (AF2 Extra MSA stack) is also directly relevant: when `D` (cell count) is bounded ~100, full column attention is fine; if we ever scale `D` to 1000s of cells, the global pooling trick is the natural fallback.

### 5.3 Global column attention (Suppl. Alg. 19) — Extra MSA Stack

The Extra MSA Stack consumes the unclustered tail of the MSA (up to `max_extra_msa = 1024` sequences). Full column attention at that depth would cost `O(1024² · N_res · c_e)` per block, which is too expensive even with only 4 blocks. AF2 substitutes a cheaper module:

**Pseudocode (paraphrased from Suppl. Alg. 19):**

```
Input  : m ∈ R^{N_seq × N_res × c_e},  c_e = 64
Output : Δm ∈ same shape

m ← LayerNorm(m)

# Single-head K and V (no head dimension on K, V).
k_{si,c}    = Linear(m_{si})      shape (N_seq, N_res, d)
v_{si,c}    = Linear(m_{si})      shape (N_seq, N_res, d)
# Multi-head Q, but pooled across sequences first.
q_pool_{i,h,c} = Mean_s [ Linear(m_{si}) ]  → mean across sequences
                                  shape (N_res, H, d)

# Gate is per-(s, i, h).
g_{shic} = sigmoid(Linear(m_{si}))   shape (N_seq, N_res, H, d)

# Attention scores are NOT per-pair. There is one "global"
# query vector per (residue, head), and N_seq key vectors per residue.
a_{ish}   = softmax_s( (1/√d) · Σ_c q_pool_{i,h,c} · k_{si,c} )
o_{i,h,c} = Σ_s a_{ish} · v_{si,c}

# Gate, then project back to c_e.
weighted_avg_{i,h,c} := o_{i,h,c}            # broadcast over s next
Δm_{s,i} = Linear_out( concat_h( g_{shic} ⊙ weighted_avg_{i,h,c} ) )
```

**What changes vs. full column attention.**

1. **Queries are mean-pooled across sequences before projection.** So there is exactly one query vector per (residue, head), not one per (sequence, residue, head). This collapses the attention map from `(N_seq × N_seq)` per column to `(1 × N_seq)` per column.
2. **K and V are single-head** in the AF2 codebase (`GlobalAttention.k_weights` has shape `(c, d)`, no head axis). The multi-head dimension is then re-introduced by broadcasting the gate over heads in the output.
3. **Gating is still per-(s, i, h).** This is the only place each sequence's identity comes back into the output. Without per-row gating, every row would receive the same residual update at residue `i` (the weighted average is the same across `s`). The gate makes the output sequence-specific.
4. **Cost.** O(N_seq · N_res · c_e) for the attention scores, vs. O(N_seq² · N_res · c_e) for full. With N_seq = 1024 that is a 1024× reduction in the score-computation step.

This trade-off — keep gating sequence-specific but collapse the score matrix to one global query — is the AF2-specific knob for handling large MSAs. The MSA Transformer paper (Rao et al. 2021) used full row + full column attention everywhere; AF2 added the global variant for the Extra MSA stack.

**For our scRNA case:** If we ever wanted to keep D ≫ 100 cells, the AF2 Extra-MSA recipe says: a single learned-pooled query per genomic bin, single-head K/V, per-cell sigmoid gate. The information bottleneck this creates — a single content vector per bin summarizing the whole cell population — is exactly the type of bottleneck our `CellAxisPool` from `7_axial_attention_ntv3_integration.md` §3.4 already implements, just with `n_queries = 1` and without the per-cell residual.

[CONFIRMED — Suppl. Alg. 19, repo `modules.py:MSAColumnGlobalAttention`, `modules.py:GlobalAttention`]

### 5.4 What column attention does NOT do

- It does **not** mix information across different residues. Two MSA cells at different columns never interact inside this module.
- It does **not** see the pair representation. The pair-bias term only enters row attention.
- It does **not** use a positional encoding on the sequence axis. The set of MSA rows is treated as unordered. Row 0 is special only because it carries the target sequence's features through the input embedder, not because the attention module knows it is row 0.
- It does **not** symmetrize the s axis. Column attention is plain self-attention (not "tied" across columns in any way), unlike GPN-Star's sequence-wise attention, which ties Q/K to the primary target across all rows.
- It does **not** have a triangle-attention-style geometric inductive bias. That bias lives entirely in the pair representation update.

[CONFIRMED — Suppl. §1.6, repo `modules.py`]

## 8. Concrete comparison: AF2 column attention vs. our planned cell-axis attention

| Property | AF2 full column attn | AF2 global column attn | `7_axial_attention_ntv3_integration.md` §3.3 |
|---|---|---|---|
| Vertical axis | MSA sequences `N_seq` ≤ 512 | MSA sequences `N_seq` ≤ 1024 | Cells `D` ≤ 100 |
| Horizontal axis | Residues `N_res` (typically 256 in training) | Residues `N_res` | 128-bp bins `T` (256 for 32-kb window) |
| Channels | `c_m = 256` | `c_e = 64` | `d_cell = 256` |
| Heads | 8 | 8 (gate); 1 head for K/V | 8 |
| PE on vertical axis | None | None | None |
| PE on horizontal axis | (column attn has none; row attn is RoPE-free, uses relative position via pair repr) | Same | RoPE on row attention |
| Cost per layer | O(N_seq² · N_res · c) | O(N_seq · N_res · c) | O(D² · T · d_cell) |
| Mixes vertical and horizontal in same module? | No (axial). Separate row + column. | No (axial). | No (axial). |
| Gating | Yes (sigmoid on attn output) | Yes | Optional; recommended once block depth >2 |
| Pair representation? | Yes (separate `z` tensor, updated via outer-product mean + triangle ops) | Yes (shared with main Evoformer) | Not in v0; could add a bin × bin pair tensor for promoter-enhancer covariation later |
| Number of stacked blocks | 48 | 4 | 4 |

The structural prescription that AF2 confirms for us:

1. **Keep the 2-D MSA tensor intact across the encoder.** Don't flatten the cell axis early.
2. **Use plain self-attention on the cell axis.** No positional encoding, no clade structure, no bias term — cells are an unordered set.
3. **Mix row and column on alternating sub-blocks, not jointly.** Don't try to do full 2-D attention; the axial decomposition is what makes the cost tractable.
4. **Gate the output of each axial attention.** It costs little and gives the model a per-row escape valve.
5. **If `D` ever grows beyond ~512, replace full column attention with the AF2 global variant** (pooled query, single-head K/V, per-row gate).

## 9. Differences from MSA Transformer (Rao et al. 2021)

MSA Transformer is the direct precursor. Key differences relevant to column attention:

| Property | MSA Transformer | AF2 (main Evoformer) |
|---|---|---|
| Backbone task | MLM on MSAs (pretraining) | Structure prediction |
| MSA axes | (N_seq, N_res, c) | Same |
| Row attention | "Tied row attention" — single attention map shared across rows; ungated | Per-row gated attention biased by pair repr |
| Column attention | Plain multi-head self-attention along sequences | Same. AF2 inherits column attention essentially unchanged. |
| Pair representation | None — there is just the MSA repr | Separate pair repr maintained throughout |
| Number of layers | 12 | 48 (+ 4 Extra MSA blocks) |
| Extra MSA / global variant | None | Yes, used for the unclustered tail |

So AF2's column attention is the **same** mechanism as MSA Transformer's column attention. The novelty of AF2 (relative to MSA Transformer) is the row-attention pair bias, the pair representation, and the triangle ops — none of which are in the column-attention module.

[CONFIRMED — Rao et al. 2021, AF2 Suppl. §1.6]
