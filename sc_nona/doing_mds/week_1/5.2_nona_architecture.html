Nona: a multimodal masking framework for functional genomics

One model. One transformer. Many tasks. Nona ingests DNA sequence and base-resolution functional tracks side-by-side, masks arbitrary subsets of either modality, and asks the network to reconstruct the missing values. The choice of mask defines the task.

The one-paragraph summary

Nona is a Borzoi-style hybrid conv-transformer U-Net, but with a twist: both DNA and functional tracks are both inputs and outputs, gated by binary masks. Mask all tracks → it's a sequence-to-function model (Enformer). Mask the central window of tracks → it's a context-aware extrapolator. Mask the sequence → it's a functional-context language model. Mask reads at SNPs → it does genotyping. One architecture, one weight set per model, many heads emerge from masking alone.

§ 1 — The input/output contract

Two parallel tapes, four masks

The model sees the genome as a stack of co-registered tracks: one is DNA, the rest are functional. Everything is base-resolution, everything is indexed by the same genomic coordinate, and any position in any channel can be masked.

Mask values are per-position, not per-channel: all 4 sequence channels at a position share the same mask bit. The model has to use the mask to tell apart "genuinely missing base (N)" from "intentionally hidden", and real zeros in a track from masked zeros.
Why two parallel tapes, not MSA-style stacking? The vertical axis is modality (sequence + T fixed bulk tracks), not homologous samples. There is no cross-attention between tracks — they are fused early into a single representation by elementwise summation after their separate conv encoders. Adding more tracks ≠ adding rows of an alignment; it's adding channels.
§ 2 — Architecture

Hybrid conv-transformer U-Net, simplified from Borzoi

Separate convolutional encoders embed sequence and tracks independently. Their representations are summed just before the transformer. The transformer does the long-range work. A symmetric upsampling decoder, with U-Net skip connections from both encoder branches, recovers base resolution.

The fusion choice. Two separate conv encoders, summed before transformer. This is a deliberate design over (a) concatenating channels from step 1, or (b) using cross-attention between modalities. The advantage: at sequence-to-function inference (tracks are all-masked, all-zero), the model can zero out the track skip connections and the deepest track representation to avoid batch-norm instabilities — a trick the paper calls out explicitly.

Two configurations of the same architecture. The context-aware model uses n_reductions=7 (full U-Net, downsamples 128×). The functional language model and genotyping model use n_reductions=0: no downsampling, no skips needed, pure conv+transformer at base resolution. Same code path, different hyperparameter.
§ 3 — The masking framework

The mask is the task

Four binary masks per training example — two for inputs (what the model sees) and two for outputs (where the loss is computed). Every existing genomic foundation-model objective falls out of this by choosing the right masks.

Sequence-to-function Enformer-style

All tracks hidden, sequence fully observed. Predict tracks from sequence alone.

input_seq_mask
all observed
input_track_mask
all masked
output_track_mask
loss everywhere

Context-aware fine-tuning novel

Sequence fully observed. Tracks visible in the flanks, masked in the central 4 kb window — the same window the model must predict. Forces the model to use long-range context.

input_seq_mask
all observed
input_track_mask
center 4 kb hidden
output_track_mask
loss only in center

Functional language modeling novel

Tracks fully observed, sequence partially masked. The model must reconstruct DNA from surrounding sequence and functional context. A masked-language-model where signal tracks are an auxiliary input.

input_seq_mask
~15% positions hidden
input_track_mask
all observed
output_seq_mask
loss on hidden positions

Functional genotyping application

Track observed everywhere, sequence masked at a SNP position. Predict the most likely base at the SNP given local ATAC-seq read depth — i.e. recover genotype from chromatin accessibility.

input_seq_mask
SNP positions hidden
input_track_mask
all observed
output_seq_mask
loss at SNP only
Static vs. dynamic masking. Sequence-to-function uses a static mask: tracks are always fully hidden. The masked-language-modeling objectives use dynamic masks: each example draws a fresh random subset to hide, so the same locus produces different training signal across epochs. Validation uses fixed masks for reproducibility.
§ 4 — Losses

Cross-entropy for bases, Poisson for tracks

Two heads, two losses, one scalar weight to balance them. The output masks decide which positions count.

Sequence loss — masked cross-entropy

Averaged over positions where (a) the reference is a real base (not "N") and (b) the output sequence mask is 1.

L_seq = seq_weight · Σ CE(o, y) / |unmasked|

Track loss — masked Poisson NLL

Predictions are on log scale, so log_input=True. Per-track weights can rescale importance.

L_trk = Σₜ track_weight[t] · PoissonNLL(oₜ, yₜ) · mask
# the two loss functions, simplified from the paper def sequence_masked_loss(o, y, mask, seq_weight): # o, y, mask : B × 4 × L_out per_pos_mask = mask.any(1) & y.any(1) # positions to score loss = CrossEntropy(o, y.argmax(1), reduction="none") loss[~per_pos_mask] = 0 loss = loss.sum(-1) / per_pos_mask.float().mean(-1).clamp_min(1e-6) return seq_weight * loss.sum() def poisson_masked_loss(o, y, mask, track_weights): # o, y, mask : B × T × L_out (o on log scale) loss = poisson_nll_loss(o, y, log_input=True, reduction="none") loss[~mask] = 0 loss = loss.sum(-1) / mask.float().mean(-1).clamp_min(1e-6) return (loss.sum(0) * track_weights).sum()
Output clipping matters. Both predictions and targets are clipped to roughly (−10, ln 5000) before the Poisson loss is evaluated. Without this, a handful of high-coverage outlier positions would dominate training. The log1p input transform plus output clipping is doing a lot of variance-stabilization work that scRNA pipelines would normally handle with arcsinh or per-cell normalization.

§ 6 — Data preprocessing

BigWigs in, BigWigs out — every track is bulk

Nona does no read alignment of its own and consumes no single-cell data. All tracks are bulk, base-resolution BigWigs downloaded already-normalized from ENCODE/FANTOM, except the ATAC track which is built from BAMs using ChromBPNet's helper script.

§ 7 — What Nona is actually doing

The reframe in one sentence

Borzoi/Enformer treat tracks as only outputs; Nona treats tracks and sequence symmetrically as a joint observation with arbitrary masks. That single change is what unlocks the other tasks.

For your scRNA-into-DNA-LM project. Nona's recipe for "give the model more than just DNA" is: stack co-registered base-resolution tracks as extra input channels, fuse them by summation after a small conv encoder, and train with masking. To extend this to scRNA, the obvious drop-in is to pseudobulk cells by cell-type or by donor and produce one BigWig channel per group — but note that Nona uses a fixed small T. Scaling T into the thousands (per-donor, per-cell-type) is not addressed and would need design work. The MSA-style "vertical attention" approach you mentioned is not what Nona does.
What's missing for an scRNA extension. No strandedness for RNA tracks (CAGE is summed unstranded). No splicing / junction representation. No per-individual track stacking. No promoter-enhancer pairing supervision. No normalization beyond depth + log1p. These are the gaps any adaptation would need to fill.
