# Studies that Aligned scRNA-seq to Per-Individual / Non-Reference Genomes

**Scope:** This report surveys *studies* (not tools) that built scRNA-seq pipelines aligning reads to per-individual or non-reference personalized genomes, with an eye toward which serve as concrete templates for the `sc_nona` project — generating per-bp coverage tracks per donor from Smart-seq2/3 reads aligned to that donor's own diploid genome.

Methodology, N-masking, WASP, g2gtools, and SNPsplit as *tools* are not re-explained; see `2.4_diploid_alignment.md` for that.

---

## Executive summary

The single sharpest take from this survey: **almost nobody has done what `sc_nona` proposes.** Per-bp coverage tracks aligned to per-donor diploid genomes for scRNA-seq is a road essentially untravelled. The closest templates split cleanly by which ingredient they bring:

1. **PRIMARY TEMPLATE — Churchill-lab Diversity Outbred mESC platform (Skelly 2020 → Aydin 2023 → Aydin 2025).** This is the *only* multi-individual study using true personalized-diploid alignment at scale (170–190 DO mouse lines). It builds each animal's diploid genome from genotype data using **g2gtools** (SNPs *and indels* physically inserted, with chain files + lifted GTFs), then aligns with bowtie + **EMASE** (probabilistic EM apportionment between haplotypes) + GBRS (sample mix-up check). This is the right shape for `sc_nona` because the user's hard requirement is fidelity to each donor's actual genotype — coordinates should be in the donor's own sequence space, indels included. The only thing this template lacks is single-cell — it's bulk RNA-seq.
2. **SECONDARY TEMPLATE — Sandberg-lab Smart-seq3 lineage (Deng 2014 → Reinius 2016 → Larsson 2019 → Hagemann-Jensen 2020).** This is the only mature, reproducible single-cell pipeline with allele-aware reference handling. But it uses **N-masked CAST × C57BL/6J + SNPsplit on a single shared reference** — designed for ASE quantitation at known het sites, not for per-donor genotype-faithful coverage. *Use this for the single-cell plumbing* (zUMIs config, STAR settings, per-cell BAM → coverage), but *don't copy the genome construction* (N-masking + SNPsplit drops indels and forces a shared coordinate frame, both of which conflict with this project's goals).
3. **NEITHER target dataset uses per-donor genomes.** Cuomo 2020 and Jerber 2021 (HipSci sc-eQTL) align with plain STAR/CellRanger to GRCh37 — no WASP, no N-masking, no g2gtools. The user's planned pipeline is a genuine methodological upgrade over both.

So `sc_nona` sits in an under-explored intersection: combine the **Churchill diploid-personalization stack** (the primary thing — gives indel-faithful per-donor genomes) with the **Sandberg single-cell plumbing** (the secondary thing — gives the per-cell zUMIs / BAM → coverage handling) on HipSci data. **Read Skelly 2020 first** for the multi-individual diploid-genome alignment shape; then the Smart-seq3 GitHub repo for sc-specific config.

> **Update note (2026-06-22):** Earlier framing of this report had the Sandberg N-masked template as "highest relevance" and Churchill/g2gtools as "missing sc ingredient." That ranking was wrong for this project — the user wants indel-aware per-donor alignment, which is exactly what g2gtools does and what N-masking explicitly throws away. The reranking propagates through the tier table and recommendations below. See `2.4_diploid_alignment.md` for the matching pipeline rewrite.

## Cluster 1 — Sandberg-lab F1-hybrid mouse Smart-seq lineage

This lineage has the most operationally mature *allele-aware* scRNA-seq pipeline in existence. All papers use a CAST/EiJ × C57BL/6J F1 cross, so "per-individual" is really "the same F1 cross every time," but the alignment infrastructure is the closest analog to what `sc_nona` needs.

### Deng, Ramsköld, Reinius, Sandberg 2014, *Science* 343:193 (PMID 24408435; [DOI](https://doi.org/10.1126/science.1245316))

- Protocol: **Smart-seq** (predecessor of Smart-seq2). Full-length, no UMI, no 3'/5' bias. 269 single cells from CAST/EiJ × C57BL/6J preimplantation embryos.
- Individuals: pooled embryos from this single F1 cross; not "per-donor."
- Alignment: TopHat to mm9. Authors explicitly observed reference bias — "TopHat detected more FVB than CAST strain bias genes, likely due to an alignment bias, as the FVB strain is more closely related to the C57BL/6J reference." So this paper essentially *discovered* the bias problem that the rest of the lineage solves.
- Genotype source: Mouse Genomes Project CAST/EiJ SNPs (no per-mouse genotyping).
- BigWigs: not released.
- **Verdict:** Historical. Not a template — it is what subsequent work corrects. Useful only to motivate why the reference bias problem matters.

### Reinius, Mold et al. 2016, *Nat Genet* 48:1430 ([DOI](https://doi.org/10.1038/ng.3678); GitHub: RickardSandberg/Reinius_et_al_Nature_Genetics_2016)

- Protocol: Smart-seq2 on clonal primary mouse fibroblasts (CAST × C57BL/6J F1) and human CD8+ T cells.
- Individuals: again, one F1 cross + clonally derived fibroblasts; not per-individual.
- Alignment: this is the first paper in the lineage to do it properly — N-masked F1 genome alignment. (Confirmed by Sandberg-lab subsequent docs; full methods paywalled.)
- BigWigs: not released; only per-allele count tables.
- **Verdict:** Mid-relevance. Establishes the methodological pattern but on a single mouse cross. The GitHub repo `RickardSandberg/Reinius_et_al_Nature_Genetics_2016` is a concrete code template.

### Larsson et al. 2019, *Nature* 565:251 ("Genomic encoding of transcriptional burst kinetics"; PMC7610481)

- Protocol: Smart-seq2, 224 primary mouse fibroblasts from CAST/EiJ × C57BL/6J F1 cross.
- Alignment: N-masked F1 reference, post-hoc allele assignment by SNP overlap. Same pattern as Reinius 2016 (extends it).
- BigWigs: not released. Allele-resolved counts only.
- **Verdict:** Mid-relevance. Same template as Reinius 2016, slightly larger scale (224 cells). Notable as a successful downstream science output of the pipeline (per-allele burst kinetics).

### Hagemann-Jensen, Ziegenhain, ..., Sandberg 2020, *Nat Biotechnol* 38:708 — Smart-seq3 (PMID 32518404; [DOI](https://doi.org/10.1038/s41587-020-0497-0))

- Protocol: **Smart-seq3** — full-length + 5'-UMI hybrid. ~60% of molecules assignable to allele of origin (this is the actual selling point of Smart-seq3 vs Smart-seq2).
- Individuals: one F1 cross again (CAST × C57BL/6J) for mouse work; also human PBMCs from a single donor.
- Alignment: **zUMIs pipeline → STAR → N-masked CAST/EiJ × C57BL/6J genome.** The `mouse_cross.yaml` config in `sandberg-lab/Smart-seq3/allele_level_expression/` makes this explicit:
  ```
  STAR_index: /mnt/storage1/genomes/Mouse_CAST_Nmasked/STAR5idx_noGTF/
  GTF_file:   /mnt/storage1/genomes/Mouse/Mus_musculus.GRCm38.91.chr.clean.gtf
  ```
  The N-masked FASTA is built with SNPsplit's `SNPsplit_genome_preparation` from a CAST/EiJ-specific dbSNP142 VCF, filtered for variants seen in CAST × C57BL/6J F1 data. Allele assignment is a post-hoc R script (`get_variant_overlap_CAST.R`) on the zUMIs BAM + the CAST SNP VCF.
- BigWigs: not released, but the alignment BAMs are STAR output, so BigWigs are one `bamCoverage` call away.
- **Verdict (revised): RELEVANT FOR SC PLUMBING, NOT FOR GENOME CONSTRUCTION.** The `zUMIs` YAML + STAR config is the most directly liftable *single-cell* infrastructure. But the genome-construction half (N-masked single reference + SNP-only handling) is the wrong tool for this project — it drops indels and locks output into a shared coordinate frame. The right pattern is: take Smart-seq3's per-cell zUMIs → STAR → BAM → coverage plumbing, but swap the N-masked CAST×B6 reference for a g2gtools-built per-donor diploid FASTA (Churchill template, Cluster 2). The `get_variant_overlap_CAST.R` allele-tagging script is also not relevant when reads are already aligned to physically-separate haplotype contigs.

### Sandberg-lineage follow-ups (2020-2026)

The 2022 Smart-seq3xpress paper (Hagemann-Jensen et al., *Nat Biotechnol*) is a throughput improvement, not an alignment-strategy change. Larsson, Coucoravas, Sandberg, Reinius 2019 *Nat Struct Mol Biol* (PMID 31582851) on X-chr burst frequency reuses the same Smart-seq2 + N-masked F1 pipeline. Vértesy et al. 2018 *Nat Commun* (PMID 29760424) is an interesting outlier — they used **exome-sequencing-derived parental haplotypes** on five human fetal donors and assigned scRNA-seq reads to maternal/paternal alleles. That is, they did per-donor haplotype phasing with WGS-equivalent info, then allele-tagging on scRNA-seq. Smaller scale (5 donors, ~300 cells) but methodologically interesting; the haplotype-aware approach is closer to a "per-donor diploid" pipeline than the rest of the lineage. Worth a closer look as a second template.

## Cluster 2 — Churchill-lab Diversity Outbred (DO) mouse genetics platform

This is the **only mature multi-individual personalized-diploid alignment pipeline at scale** that I could find. Every DO mouse is a unique 8-way founder mosaic; the pipeline reconstructs each mouse's personal genome from genotype data.

### Skelly et al. 2020, *Cell Stem Cell* 27:459 ("Mapping the Effects of Genetic Variation on Chromatin State and Gene Expression Reveals Loci That Control Ground State Pluripotency"; PMC7484384)

- Protocol: **bulk RNA-seq** (not single-cell) + bulk ATAC-seq on 185 DO mESC lines.
- Number of individuals × cells × tissues: 185 DO mice × 1 cell type (mESC) × bulk. No single cells.
- **Alignment strategy: this is the canonical g2gtools/EMASE/GBRS pipeline.**
  - "Constructed strain-specific genomes for the eight inbred founder strains using the software tool g2gtools and genetic variation data from Mouse Genomes Project SNP and indel release."
  - Aligned PE reads with **bowtie v1.1.2** to a pooled "8-way" transcriptome containing strain-specific isoform sequences from all 8 DO founders.
  - **gbrs v0.1.6** for genotype inference + sample mix-up checking from RNA-seq.
  - **EMASE v0.10.16** to resolve multi-mapping and apportion reads probabilistically across founder alleles.
- Genotype source: GigaMUGA SNP array on each DO mouse, used to infer haplotype blocks across the 8 founders.
- BigWigs: not explicitly released; raw data on ArrayExpress E-MTAB-7728; processed data and code on GitHub (`daskelly/CellStemCell_2020_diverse_mESCs`).
- **Verdict (revised): HIGHEST RELEVANCE — the primary template for this project.** This is the only published multi-individual personalized-*diploid* RNA-seq pipeline at scale, and crucially it handles **indels** (via g2gtools materializing them in each personal FASTA) — which is exactly the property the user has hard-locked in. The bulk-vs-single-cell gap is the easier half to close (lift the per-cell zUMIs/STAR plumbing from the Sandberg Smart-seq3 repo); the indel-aware diploid-personalization half is the hard part, and Skelly 2020 already solved it. Methods section + `daskelly/CellStemCell_2020_diverse_mESCs` GitHub repo are the right things to read first.

### Aydin et al. 2023, *Cell Genomics* 3:100283 (pluripotent proteome of DO mESCs; PMC10112288)

- Same lineage as Skelly 2020. Same pipeline: bowtie v1.1.2 → 8-way pooled founder transcriptome → EMASE/GBRS. 190 DO mESC lines. Bulk RNA-seq.
- Adds proteomics; alignment side is unchanged.
- **Verdict:** Same template as Skelly 2020.

### Aydin et al. 2025, *G3* 15:jkaf099 (DO mNPC; PMC11785210)

- 186 DO mouse neural progenitor cell lines, **bulk** RNA-seq, same bowtie → 8-way pooled transcriptome → EMASE/GBRS pipeline.
- **Verdict:** Confirms the pipeline is the lab's stable production stack. Same template as Skelly 2020.

### scRNA-seq from this lab?

I could not find a published *single-cell* RNA-seq paper from the Churchill / Munger / Skelly groups using g2gtools + EMASE on DO mice. The 2024 X-aneuploidy preprint (Aydin et al., bioRxiv 2024.06.29.601344) mentions two scRNA-seq datasets on DO mESCs, but I could not access the methods (paywall + 403s). If that work uses the personal-genome pipeline at single-cell, **it would be the closest existing analog to `sc_nona`** and is worth chasing down directly (the user has bioRxiv access).

### Other DO/CC mouse sc papers

There is a 2025 study (Olivieri et al., G3 / bioRxiv) that cultured BMSCs from ~80 DO mice and ran scRNA-seq for bone-mineral-density network analysis, but the methods I could see do not mention g2gtools/EMASE — likely standard CellRanger to mm10. Not a template.

## Cluster 3 — Other F1-hybrid sc studies

Smaller / scattered:

- **Vértesy et al. 2018, *Nat Commun* (PMID 29760424).** Already mentioned above. Five human female fetal donors; matched exome of fetus and mother to phase parental haplotypes; Smart-seq2 scRNA on ~300 germ cells; reads assigned to maternal/paternal allele via overlap with phased SNPs. The aligner / N-masking detail I could not extract from the abstract — but the *concept* (per-donor phased haplotypes + sc allele tagging) is genuinely closer to `sc_nona`'s setup than the F1-mouse work. **Worth reading the methods.**

- **F1 hybrid mouse macrophage paper, 2025 *Sci Reports* (s41598-025-21643-w).** Reported as cis-eQTL on F1 macrophages using "an allele-separated pseudogenome approach." Paywall blocked methods extraction. The pseudogenome approach is dual-FASTA-like; relevance medium.

- **Sandberg-lab Smart-seq3 human PBMC** (Hagemann-Jensen 2020) uses a single human donor with a phased VCF; allele assignment then works the same way as the mouse pipeline. This is the closest *human* application of the Sandberg lineage and confirms the pipeline transfers cross-species.

No mass-population F1 study with full base-pair coverage release exists that I could find.

## Cluster 4 — Population-scale sc-eQTL studies (per-donor genomes?)

The big-picture finding here is **none of the major sc-eQTL atlases use per-donor genome alignment.** They all align to a single reference with CellRanger / STAR, demultiplex donors via shared variants (demuxlet / Vireo), and treat genotype information separately at the QTL-mapping stage.

### Cuomo et al. 2020, *Nat Commun* 11:810 (PMID 32041960; [DOI](https://doi.org/10.1038/s41467-020-14457-z))

- Protocol: Smart-seq2 on 125 HipSci iPSC donors × ~5 days of endoderm differentiation. ~36k single cells.
- **Alignment: plain GRCh37 + STAR in two-pass ENCODE-default mode. NO WASP. NO N-masking. NO personalized genome.** ASE counting done post-hoc with GATK `ASEReadCounter -minDepth 1 -minMappingQuality 10`.
- Genotype source: HipSci WGS, SHAPEIT-phased, IMPUTE2-imputed.
- BigWigs: not released; only count matrices on Zenodo (the `ase_aggregated_by_donor_open_access_lines.tar.gz` from this paper is the ASE table).
- **Verdict:** This is the user's *target dataset* and a methodological floor — what they will exceed. Not a template.

### Jerber et al. 2021, *Nat Genet* 53:304 (PMID 33664506; [DOI](https://doi.org/10.1038/s41588-021-00801-6))

- Protocol: **10x Chromium** (3' biased — does not meet the user's purity criterion). 215 HipSci iPSC lines → midbrain/dopaminergic neurons. >1 million cells.
- **Alignment: CellRanger v2.1.0 → GRCh37/hg19. No WASP, no N-masking, no per-donor genomes.** Donor demultiplexing via demuxlet on shared exonic variants.
- Genotype source: HipSci genotyping/WGS.
- BigWigs: not released.
- **Verdict:** Same as Cuomo — a methodological floor, not a template. Also 3'-biased and so not suitable as the actual data source for the user's project, only as a population-scale companion.

### Yazar et al. 2022, *Science* 376:abf3041 — OneK1K (PMID would be ~35389781)

- Protocol: 10x 3' on PBMCs from 982 donors. 1.27M cells.
- Alignment details paywalled but every secondary description says CellRanger to standard reference. No personalized genome. Genotyping array + imputation for genotype data.
- **Verdict:** Same — population-scale ASE atlas built on top of *non-personalized* alignment. Not a template.

### TenK10K (Xue et al. 2025 medRxiv / Powell lab) — caQTL/multiome atlas

- 1042 donors, 3.5M nuclei scATAC-seq + 90 multiome. WGS-matched. The pipeline (powellgenomicslab/tenk10k_phase1_multiome) appears to use standard CellRanger-ATAC + CellRanger-ARC to GRCh38; the caQTL mapping is done post-alignment. I could not confirm whether they apply WASP on the scATAC side, but the multiome paper does not describe per-donor genome construction.
- **Verdict:** Population-scale precedent for sc-genomic-variation work, but not a per-donor genome template.

### sc-eQTLGen consortium (van der Wijst, Westra, Franke et al.)

- The WG0/WG1 pipeline (sc-eQTLGen-consortium on GitHub) is the closest thing to a standardized sc-eQTL preprocessing pipeline. Reading the WG1 docs (wg1-pipeline-qc.readthedocs.io), the alignment step is **standard CellRanger to GRCh38** — no per-donor genome construction. They use Souporcell / Vireo for donor demultiplexing.
- **Verdict:** Not a template. But the documentation is a good model of how to *standardize* a multi-cohort sc pipeline.

### Sarkar / iPSCORE — Sarkar, Gilad et al. 2019 *PLoS Genet* (vQTL paper)

- 53 Yoruba donors × Fluidigm C1 / Smart-seq2-like scRNA-seq, ~5447 cells.
- Alignment described as standard reference + WASP-style filtering in some downstream analyses; no per-donor diploid genome construction.
- **Verdict:** Medium relevance. Smaller than HipSci, but the only population-scale sc study besides HipSci that uses full-length sc (not 3'). Worth a quick look — but not a per-donor genome template.

## Cluster 5 — Single-cell ATAC + personal genomes

Several relevant population-scale scATAC studies (TenK10K, immune caQTL studies summarized in medRxiv 2024.12.05.24318552) but **none use per-donor genome alignment**. They all align to GRCh38 with CellRanger-ATAC; some apply WASP at the BAM stage; QTL mapping is downstream. Skelly 2020 again is the closest analog (bulk ATAC + bulk RNA on DO mice with g2gtools personalization). No published scATAC paper applies g2gtools-style per-individual alignment that I could find.

**Verdict:** Cluster is empty as templates. Useful only as a confirmation that the sc-genomic-QTL field has not adopted personalized alignment — `sc_nona`'s approach is novel here too.

---

## Cluster 6 — Pangenome / graph-genome sc alignment (2023–2026)

The personalized-pangenome and impute-first literature (Sirén et al. 2024, *Science*; Mun et al. 2024, *Genome Biology*) explicitly addresses per-individual diploid alignment for *short-read DNA-seq*. Both papers note that RNA-seq application is unestablished. I could find **zero published sc/single-cell pangenome alignment papers** as of mid-2026. The closest is the smartSim simulation tool for splice-aware Smart-seq3 data (Oxford Bioinf Adv 2025), which is about benchmarking, not personalized alignment.

**Verdict:** Cluster is essentially empty. `sc_nona` would be early to this space if it went the graph-genome route — which probably means it shouldn't, because the tooling isn't there yet. Stick with N-masked diploid + WASP.

## Recommendations — which methods sections should the user read first?

(Revised priority: the *diploid personalization* half is the harder problem and the primary thing to learn from; the *single-cell* half is the easier add-on.)

1. **Skelly et al. 2020 Cell Stem Cell — "Mapping the Effects of Genetic Variation on Chromatin State and Gene Expression Reveals Loci That Control Ground State Pluripotency"** (PMC7484384) — **primary template.** This is the only published multi-individual personalized-*diploid* RNA-seq pipeline at scale, and it handles indels (via g2gtools) rather than throwing them away (which is what N-masking does). Read the Methods section ("RNA-seq alignment and quantification") and pull up `daskelly/CellStemCell_2020_diverse_mESCs`. The g2gtools + EMASE + GBRS pattern is the foundation. The user adapts by (a) building two haplotype FASTAs per donor instead of an 8-way pooled founder transcriptome (HipSci has phased VCFs, not founder mosaics), (b) doing this for human not mouse, and (c) running it at single-cell resolution.
2. **`churchill-lab/g2gtools` README and `emase` docs.** The bash-level mechanics of "VCF → per-individual FASTA with indels + chain file + lifted GTF" and "diploid BAM → per-haplotype apportionment via EM." This is concretely what Step 1–4 of the revised `2.4_diploid_alignment.md` recipe needs to call.
3. **Hagemann-Jensen et al. 2020 *Nat Biotechnol* — Smart-seq3, and `sandberg-lab/Smart-seq3` GitHub** — **secondary template, for sc plumbing only.** Lift the zUMIs YAML structure, STAR per-cell args, and the per-cell BAM → coverage flow. Do *not* lift the N-masked CAST×B6 genome construction or the `get_variant_overlap_CAST.R` allele-tagging script — those are designed for a different problem (ASE on a shared reference) and conflict with per-donor diploid alignment.

If time allows, also skim:
- **Reinius et al. 2016 *Nat Genet*** (`RickardSandberg/Reinius_et_al_Nature_Genetics_2016`). Older but cleaner Sandberg-lineage code. Same caveat as Smart-seq3: lift the sc handling, not the genome construction.
- **Vértesy et al. 2018 *Nat Commun*** (PMC5951918) — per-donor phased haplotypes + sc allele tagging on 5 human donors. Conceptually closest to `sc_nona`'s setup (per-donor phasing on humans, full-length sc reads); worth a methods read for the phasing pattern.
- **Aydin et al. 2024 bioRxiv on X aneuploidy in DO mESCs** (2024.06.29.601344) — the only place in the Churchill lineage that may have applied g2gtools/EMASE at single-cell resolution. If confirmed, this would be the closest existing precedent for the full `sc_nona` shape; worth chasing the full PDF.

The user should *not* spend time reading Cuomo 2020 or Jerber 2021 methods for alignment guidance — those papers are the floor, not the ceiling. Read Cuomo 2020 for the data structure and download metadata, not the alignment.

## Notes on what's missing / flagged uncertainty

- I could not directly access several paywalled Nature / Science PDFs (Smart-seq3 main paper, Larsson 2019, Yazar 2022 OneK1K, the 2025 macrophage paper). Methods details for those papers were extracted from secondary sources, PMC mirrors where available, and the GitHub configs. The Smart-seq3 alignment details are confirmed from the open YAML config and README in `sandberg-lab/Smart-seq3` — high confidence. Larsson 2019 alignment is inferred to match Reinius 2016 — medium confidence; the paper is paywalled.
- The Aydin 2024 X-aneuploidy preprint is the single most likely place a Churchill-lab scRNA-seq + g2gtools pipeline lives, but the bioRxiv full PDF returned 403 errors. Worth a follow-up directly.
- The sc-eQTLGen WG1 pipeline alignment step is described from the secondary `wg1-pipeline-qc.readthedocs.io` site rather than the actual Snakefiles — high confidence on "no per-donor genome" but lower confidence on the exact WASP usage.
- No PubMed-indexed F1-mouse Drosophila sc paper turned up with a clear personalized-genome scRNA pipeline; some Drosophila F1 scATAC work (cited via scDALI) but on the allele-counting side, not on per-fly genome alignment.
